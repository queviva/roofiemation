roofiemation.js allows elements to animate between any state, at any moment, making css animations effectively function like transitions

[hit ctrl+space or click here to slow down the animation; hit space or click here to reset the animation to normal speed; you can also scroll around on the roof to change the color, or hit p to toggle pausing]
include the script, along with any options in a well-formated JSON data-param, then indicate which elements you want to make into roofies by giving them a data-roofie parameter; animate by calling roofie.mate() and passing in the name of a class that has the keyframe animation-name you want: 

<script src="roofiemation.js"
 data-roofie='{
    "selector" : "roofie",
    "watchers" : "transform opacity d"
 }'></script>
 
<div id="sample" data-roofie></div>

<style>

    @keyframes sampleAni {
        0% {}
        50% { ... }
        100% { ... }
    }
    
    aniClass {
        animation-name: sampleAni;
    }

</style>

<script>

    // call any of the roofie methods //
    sample.roofie.mate('aniClass');
    sample.roofie.paz();
    sample.roofie.stop();
    
</script>
what do each of these options mean?
  • selector: the string that you want to use as the data-param to label which elements should be selected; by default, roofiemation.js will select all elements with a data-roofie parameter; this is also the name of the object that will get added to any roofies - for example, if you change the selector to "animonster", then you would use obj.animonster.paz() to pause the roofiemation, and label all the roofies with data-animonster parameters

  • watchers: this is a coma separated list of all the attributes that you would like the roofiemation to remember the state of; you should list all of the style attributes that get changed during the animation; by default, roofiemation.js will preserve the memory of the transform, opacity and svg paths [... though that'll probably change, those are just the most commonly used in these demos]

  • @keyframes: the keyframe animations you create should have, obviously, no value for 0%; this way, roofiemation is able to animate from any current condition of a roofie element

  • animationClass: the roofies will animate when they are given a class that contains an animation-name

  • roofie.mate(string): is the roofie method that triggers an animation; it takes a string which is the name of a class that contains an animation

  • roofie.paz(): this will pause any roofiemation

  • roofie.stop(): this will force a roofiemation to stop

ok, so how is this achieved? let's look at an incredibly simple animation that changes the path of an svg element: 

<!-- include the script -->
<script src="roofiemation.js"></script>

<!-- create a shape with roofie param -->
<svg><path id="shape" data-roofie/></svg>

<!-- in the styles ... -->
<style>

    /* for that element ... */
    #shape {
    
        /* create several css var paths */
        --triangle : 'M0 0 ...';
        --square   : 'M0 0 ...';
        --hexagon  : 'M0 0 ...';
        
        /* assign an initial path */
        d: path(var(--square));
        
    }
    
    /* create a class for each path */
    /* all using the same animation */
    .tri {
        --path : var(--triangle);
        animation-name: shapeAnimation;
    }
    
    .sqr {
        --path : var(--square);
        animation-name: shapeAnimation;
    }
    
    .hex {
        --path : var(--hexagon);
        animation-name: shapeAnimation;
    }
    
    /* create the keyframe animation */
    @keyframes shapeAnimation {
    
        /* always have nothing at zero */
        0%{},
        
        /* change the path of the shape */
        100%{
            d: path(var(--path));
        }
        
    }
    
</style>

<script>

// listen for clicks
window.addEventListener('click', () => {

    // pick a random classname
    let rand = ['tri','sqr','hex'][~~(Math.random()*3]);
    
    // call the roofie animation method
    shape.roofie.mate(rand);

});

</script>
